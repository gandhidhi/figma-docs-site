コンポーネントはFigmaの最も重要な機能の一つです。ボタンやカードなどのUIパーツをコンポーネント化することで、デザインの一貫性を保ちながら効率的に作業できます。

インスタンスのプロパティを上書きする

インスタンスはメインコンポーネントのスタイルを継承しますが、個別に変更（上書き）できます。

上書きできるプロパティ
テキスト内容: ボタンのラベルなど
画像: アバターやサムネイル画像
色: 特定のインスタンスだけ色を変える
表示・非表示: 特定の要素を隠す

上書きをリセットする

インスタンスを右クリックして Reset all overrides を選択すると、メインコンポーネントの状態に戻せます。

バリアントの活用

バリアントを使うと、1つのコンポーネントで複数のバリエーション（状態）を管理できます。

バリアントの例

`
Button コンポーネント
├── Type: Primary / Secondary / Danger
├── Size: Small / Medium / Large
└── State: Default / Hover / Disabled
`

バリアントを作成する

コンポーネントのベストプラクティス

命名規則

`
カテゴリ/コンポーネント名/バリアント
例:
Button/Primary
Button/Secondary
Form/Input/Default
Form/Input/Error
Navigation/Tab/Active
`

コンポーネントの整理
専用ページを作る: デザインファイル内に「Components」ページを作り、全コンポーネントをまとめる
カテゴリ別にフレームで区切る: ボタン・フォーム・カードなどカテゴリごとにフレームで整理
説明を追加する: コンポーネントを選択して右パネルの「Description」に使い方を記述

デザインシステムへの発展

コンポーネントが増えてきたら、別のファイルを「コンポーネントライブラリ」として公開し、チーム全体で共有できます。
